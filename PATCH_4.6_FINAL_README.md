## Технические детали для любопытных

### Почему `outputs` теперь сохраняется
В Prisma `Workflow.graph` — это поле типа `Json`. `saveWorkflowGraph` тупо записывает в него любой JSON через `prisma.workflow.update({ data: { graph } })`. То есть бэк не различает что внутри — он просто хранит JSON. Поэтому добавив `outputs/results` в клиентский `cleaned` объект мы автоматически их сохраняем.

При загрузке `initialGraph` приходит назад из БД целиком — со всеми outputs. Кэш executor'a в `executor.ts` это уже видит и использует для subgraph runs.

### Почему edges перестали выглядеть как пауки
Изменён расчёт control offset для bezier:
```ts
// Старая формула — линейная от расстояния, без верхней границы
const dx = Math.max(40, Math.abs(p.x2 - p.x1) * 0.4);

// Новая формула — менее агрессивная и с верхней границей
const dx = Math.max(40, Math.min(140, Math.abs(p.x2 - p.x1) * 0.35));
```
При расстоянии 1000px между нодами:
- Старая формула: dx = 400px → большая S-образная кривая
- Новая формула: dx = 140px (clamp) → плавная аккуратная дуга

### Почему `key={edges-${dragTick}}` пришлось убирать
Этот трюк работал в 4.5 чтобы заставить SVG перерисоваться на каждом drag-кадре. Но с моим useLayoutEffect для DOM-измерения портов он создавал баг:
1. Drag tick меняется
2. CanvasEdges remount'ится → cache измерений пустой
3. useLayoutEffect срабатывает → измеряет порты → setState
4. setState → re-render
5. dragTick меняется снова → goto 1

Заменив `key` на обычный prop `dragTick={dragTick}` мы:
- Сохраняем re-render компонента на каждом dragTick
- Но НЕ ремаунтим — кеш `measuredRef` живёт между рендерами
- useLayoutEffect срабатывает только когда `graph.nodes / pan / zoom` меняются, и даже тогда не вызывает re-render если позиции не изменились (diff-проверка перед `setMeasureVersion`)
